# Session Kickoff Pack — Live Military Conversion Validation

Date: 2026-03-28
Purpose: Resume immediately in a new chat and validate live military non-carry-through cases with evidence-first classification.

## Mission
Validate current in-game military conversion behavior using observed non-carry-through cases.
For each reproduced case:
1. Trace every affected unit to final outcome.
2. Reconcile per-unit lines with LEAVER_END and LEAVER_AUDIT totals.
3. Classify each issue as Functional Bug, Intentional Policy, or Visibility Gap.
Do not propose code changes until classification is complete.

## Known Cases To Reproduce First
1. mongol -> french_ha_01
2. abbasid_ha_01 -> english
3. ottoman -> french_ha_01
4. hre -> english
5. malian -> chinese_ha_01 (or equivalent civ-unique fallback case)

## Required Evidence Per Case
- LEAVER_START
- Per-unit LEAVER_XFER lines
- AGE-OK / AGE-BLOCK lines
- LEAVER_END summary line
- LEAVER_AUDIT decision line
- Route and destroy bucket totals from LEAVER_END

## Pass/Fail Reconciliation Rule
For each case, unit-level outcomes must numerically match LEAVER_END and LEAVER_AUDIT totals.
If counts differ, classify as Functional Bug unless proven to be instrumentation-only.

## Classification Rules
- Functional Bug: incorrect final outcome, wrong route attribution, or counter mismatch.
- Intentional Policy: explicit and expected behavior (age-block, policy destroy, transfer-only neutral).
- Visibility Gap: behavior likely correct but logs/counters do not fully attribute outcomes.
