### 1.2 tsconfig.json — Minimal TSTL Configuration

```json
{
  "compilerOptions": {
    "target": "ESNext",
    "lib": ["ESNext"],
    "moduleResolution": "Node",
    "types": [],
    "strict": true
  },
  "tstl": {
    "luaTarget": "JIT"
  }
}
```

| Lua 5.0 | `"5.0"` |
| Universal | `"universal"` |

### 1.4 Type Declaration Strategy

- **Official Lua stdlib types:** `lua-types` npm package
- **Game-specific types:** Unofficial community packages exist for Dota 2, Factorio, WoW, Garry's Mod
- **AoE4 types:** `@aoemods/aoetypes` (see §2)
- **Custom overrides:** Create `types.d.ts` in project `scripts/` folder to override broken types
- **Any existing API** can be declared using standard TS declaration files

### 1.5 Publishing TSTL Libraries

| Setting | Value |
|---|---|
| `compilerOptions.declaration` | `true` |
| `tstl.buildMode` | `"library"` |
| `package.json files` | `["dist/**/*.lua", "dist/**/*.d.ts"]` |
| `package.json types` | `"./dist/index.d.ts"` |
| `package.json main` | `"./dist/index"` (no extension!) |

**Limitations:**
- Cannot import `.ts`/`.tsx` directly from `node_modules`
- Cannot bundle Lua libraries with `luaBundle`
- Two library types: **type declarations** (ambient types only) and **Lua libraries** (importable transpiled code)

## 2. AoE4 SCAR Integration via TSTL

### 2.1 Project Lineage

| Project | Status | Purpose |
|---|---|---|
| `aoemods/AOE4-TSTL` | **Archived** (Apr 2022) | Experimental TS→Lua for AoE4; partially autogenerated types from scardocs, most functions return `any` |
| `aoemods/aoe4-typescript-template` | **Active** (5★) | Replacement template using `@aoemods/aoetypes` npm package |

### 2.2 Active Workflow (aoe4-typescript-template)

```
npm install → set .env output path → npm run dev (watch mode transpile + copy)
```

| Command | Purpose |
|---|---|
| `npm install` | Install TSTL + aoetypes |
| `npm run dev` | Watch mode — auto-transpile + copy to mod folder |
| `npm start` | One-shot build |

### 2.3 Type Override Pattern

When `@aoemods/aoetypes` has incorrect or missing types:

1. Create `scripts/types.d.ts` in project root
2. Declare corrected signatures using standard TS ambient declarations
3. TSTL picks up the override via tsconfig `include` paths

### 2.4 AoE4 SCAR Constraints for TSTL Output

- Target Lua version must match AoE4's embedded Lua runtime (Lua 5.1-compatible, LuaJIT subset)
- Output Lua must not use features beyond what SCAR's Lua VM supports
- Global namespace pollution is risky — prefix with module tables
- No external Lua package manager available at runtime — all code must be self-contained in mod assets

AoE4's SCAR engine uses a **Lua 5.1-compatible** runtime. Key constraints:

- No integer subtype (all numbers are doubles)

- `module()`/`setfenv()`/`getfenv()` may or may not be available
- Coroutines available but not commonly used in SCAR scripts

### 4.2 LuaLS Annotation System — Complete Reference

All annotations use `---` prefix (triple-dash Lua comment).

#### Core Type Annotations

| Annotation | Syntax | Purpose |
|---|---|---|
| `@type` | `---@type TYPE` | Declare variable type |
| `@param` | `---@param name TYPE` | Function parameter (append `?` for optional) |
| `@return` | `---@return TYPE name?` | Function return type |
| `@class` | `---@class ClassName : ParentClass?` | Define class with optional inheritance |
| `@field` | `---@field [scope] name TYPE` | Class field (scope: private/protected/public/package) |
| `@alias` | `---@alias Name TYPE_DEF` | Type alias or enum-like union |
| `@enum` | `---@enum EnumName` | Mark table as enum |

#### Advanced Type Annotations

| Annotation | Syntax | Purpose |
|---|---|---|
| `@generic` | `---@generic T` | Generic type parameter (backtick capture) |
| `@overload` | `---@overload fun(PARAMS):RETURNS` | Function overload signatures |
| `@operator` | `---@operator OP(OPERAND):RESULT` | Metamethod type declarations |
| `@cast` | `---@cast varname TYPE` | Cast variable to a type |
| `@as` | `--[[@as TYPE]]` | Inline type force (single-line comment) |

#### Scope & Visibility Annotations

| Annotation | Purpose |
|---|---|
| `@private` | Private to class |
| `@protected` | Accessible by subclasses |
| `@package` | File-private scope |
| `@deprecated` | Mark as deprecated |
| `@nodiscard` | Warn if return value unused |
| `@version` | Lua version requirement |

#### Workflow Annotations

| Annotation | Purpose |
|---|---|
| `@async` | Mark function as async |
| `@diagnostic` | Toggle diagnostics per line/file |
| `@meta` | Mark file as definition-only (not real code) |
| `@module` | Simulate `require` |
| `@see` | Cross-reference |
| `@source` | Source file reference |
| `@vararg` | **DEPRECATED** — do not use |

#### Advanced Type Expressions

| Type Expression | Syntax | Example |
|---|---|---|
| Union | `TYPE1\|TYPE2` | `string\|number` |
| Array | `TYPE[]` | `string[]` |
| Dictionary | `{[string]:TYPE}` | `{[string]:number}` |
| Table | `table<K,V>` | `table<string,boolean>` |
| Table literal | `{key1:TYPE}` | `{name:string, age:number}` |
| Function | `fun(PARAM:TYPE):RETURN` | `fun(x:number):string` |

### 4.3 EmmyLua Compatibility Note

LuaLS annotations were originally based on EmmyLua but **diverged at v3.0.0**. The two systems are **no longer cross-compatible**. EmmyLua is IntelliJ-focused; LuaLS is VS Code/LSP-focused. For VS Code + AoE4 modding, use **LuaLS annotations exclusively**.

## 7. Lua Style Conventions

### 7.1 Olivine-Labs Style Guide — Key Rules

Source: `Olivine-Labs/lua-style-guide` (546★)

| Rule | Convention |
|---|---|
| **Indentation** | 2-space soft tabs |
| **Scope** | Always use `local` |
| **Naming — variables** | `snake_case` |
| **Naming — functions** | `snake_case` |
| **Naming — classes/factories** | `PascalCase` |
| **Strings** | Single quotes `'...'` |
| **Line length** | 80 characters max |
| **Tables** | Constructor syntax `{}` |
| **Methods** | Use `self` for method tables |
| **Functions** | Use `function` syntax over variable syntax |
| **Returns** | Early returns preferred |
| **Booleans** | `is`/`has` prefix for boolean functions |
| **Modules** | Return table or function; no globals; file is a closure; modules are singletons |

### 7.2 Project Structure Convention

```
project/
├── src/          # Source code
├── spec/         # Tests (busted framework)
├── bin/          # Executables
├── .luacheckrc   # Linter config
└── *.rockspec    # Package spec
```

### 7.3 SCAR-Specific Deviations

AoE4 SCAR scripts typically deviate from the above:
- **Globals are standard** — SCAR expects callbacks and rule functions as globals
- **No module return** — SCAR loads files via `import` or `Scar_DoFile`, not `require`
- **Naming** — Relic convention uses `PascalCase` for functions (`Mod_Initialize`, `Rule_MyRule`)
- **file scope** — Each SCAR file's local scope is isolated per `import` call

### 8.2 middleclass — API Overview

```lua
local class = require 'middleclass'

local Fruit = class('Fruit')

function Fruit:initialize(sweetness)
  self.sweetness = sweetness
end

Fruit.static.sweetness_threshold = 5  -- class variable

function Fruit:isSweet()
  return self.sweetness > Fruit.sweetness_threshold
end

local Lemon = class('Lemon', Fruit)   -- subclassing

function Lemon:initialize()
  Fruit.initialize(self, 1)           -- explicit super call
end
```

**Install:** Copy `middleclass.lua` to `lib/`. `local class = require 'middleclass'`

**Testing:** Uses busted framework.

### 8.3 classic — API Overview

```lua
Object = require "classic"

Point = Object:extend()

function Point:new(x, y)
  self.x = x or 0
  self.y = y or 0
end

Rect = Point:extend()

function Rect:new(x, y, width, height)
  Rect.super.new(self, x, y)   -- super via .super
  self.width = width or 0
  self.height = height or 0
end

print(Rect():is(Point))  -- true (type check)
```

**Mixins:** `Point:implement(MixinClass)` — copies methods from mixin.

**Key trait:** Absolute minimalism — single file, ~40 lines, zero dependencies.

### 8.4 30log — API Overview

30 Lines Of Goodness — named/unnamed classes, single inheritance, metamethods, basic mixins.

- Variants: `30log.lua` (minified 30 lines), `30log-clean.lua` (readable), `30log-global.lua` (sets global `class`), `30log-singleton.lua` (singleton pattern)
- Class-Commons compatible via `30log-commons.lua` plugin
- Installable via LuaRocks: `luarocks install 30log`

### 8.5 SCAR OOP Relevance

AoE4 SCAR does **not** use class-based OOP natively. Patterns found in Relic's code:
- **Table-as-object:** Functions operating on shared state tables
- **Prefixed globals:** `Mod_Initialize`, `Mod_OnEvent` — flat namespace, no inheritance
- **No `require`:** Files loaded via `import`/`Scar_DoFile`, so regular Lua modules don't work as-is

**If adopting an OOP library for SCAR mods:**
- **classic** is the safest choice — tiny, no dependencies, no global pollution
- The module must be loaded via `Scar_DoFile` or embedded inline
- Avoid libraries that rely on `require` or `package.path` modification

### 9.3 Portable Patterns for SCAR

**Timer pattern** — HUMP's `timer.lua` uses `timer:update(dt)` with callbacks. In SCAR, the equivalent is:
- `Rule_AddInterval(callback, interval)` for recurring
- `Rule_AddOneShot(callback, delay)` for one-time delayed calls

**Signal/Slot pattern** — HUMP's `signal.lua` implements pub/sub with `signal.register(name, fn)` / `signal.emit(name, ...)`. In SCAR, the equivalent is:
- `Rule_AddGlobalEvent(callback, GE_EVENT_NAME)` for game events
- Custom event dispatchers can adopt the same pattern

**Gamestate pattern** — State machine with `enter`/`leave`/`update` lifecycle. Directly applicable to SCAR phase management (setup → playing → win/loss).
