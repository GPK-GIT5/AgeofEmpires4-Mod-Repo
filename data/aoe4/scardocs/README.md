## Relationship to Workspace References

The workspace `references/api/` files contain a **superset** of the engine API (~4,435 functions vs ~1,983 here) because they include gameplay helper functions extracted from Relic's SCAR scripts at runtime.

These scardocs provide information **not available** in the runtime dump:
- **Typed parameter signatures** (`PlayerID`, `ScarEntityPBG`, `Position`, etc.)
- **Function descriptions** (one-line English descriptions)

The extraction script `scripts/extract_scardocs.ps1` parses these files into structured JSON and cross-references them with the workspace docs.

## Parsed Outputs

After running `scripts/extract_scardocs.ps1`:
- `parsed_functions.json` — Structured function data with namespace, params, and descriptions
- `parsed_constants.json` — Constants grouped by prefix category

## Auto-Generated

Both `.api` files are auto-generated by XSL stylesheets (`WSCITE_API.XSL` and `WSCITE_API_ESSENCE_CONSTANTS.XSL`). They should be re-extracted after game patches that change the SCAR API.
